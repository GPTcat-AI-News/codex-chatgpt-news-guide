# 豆包、千问、元宝集体下架智能体——国内 AI 平台怎么了

> 2026年7月，字节豆包、阿里千问、腾讯元宝几乎同时下架了部分智能体功能。这不是巧合。

---

## 发生了什么

知乎热榜 253 万热度：豆包、千问、元宝集体下架智能体。

三家大厂，在同一个时间窗口，做了同一个动作。不管具体原因是合规、监管还是产品调整，这种"同时性"本身就是信号。

---

## 为什么值得关注

智能体（AI Agent）是当前 AI 行业最热的方向。OpenAI 的 Codex、Anthropic 的 Claude Code，本质都是 Agent 产品——让 AI 不只是回答问题，而是帮你执行任务。

国内平台也在跟。豆包、千问、元宝都上线过各自的智能体功能。现在集体下架，可能的原因：

- **监管合规**：智能体涉及自主执行、调用外部工具，风险比纯聊天高一个量级
- **产品质量**：仓促上线的 Agent 如果频繁出错，损伤品牌
- **政策窗口**：等待相关政策明确后再重新上线

这里有个关键区别要搞清楚：**聊天和执行是两回事。** 聊天只生成文字，最多说错话；Agent 是执行动作——调接口、读文件、操作工具。一旦出问题，不是"说错话"的级别，是"做错事"的级别。监管对后者的容忍度天然更低。

回顾 2023 年大模型刚出来的情况：当时也是一波"百模大战"疯狂上线，然后监管跟上，一批没拿到备案的直接关停。智能体现在走的是同一条路——先野蛮生长，再统一收编。

## 对不同用户的影响

**普通聊天用户：没影响。** 你平时用豆包/千问聊天、写东西、问问题，这些功能照常。下架的是智能体，不是整个平台。

**轻度 agent 用户：暂时不方便。** 如果你之前在用某个智能体做定时摘要、自动整理文件之类的事，现在得手动做，等它恢复。

**重度 agent 用户/开发者：需要找替代。** 如果你的工作流已经依赖 agent 能力（工具调用、多步执行、自动化流程），国内平台短期内指望不上。海外的 Codex 和 Claude Code 不受这波影响，但国内直接访问有门槛。

替代方案：

- 日常 AI 使用（聊天/写作/代码）：国内聚合平台仍然可用，如 [ZEOGPT](https://www.zeogpt.com/register?ref=Ac3KbS3F)
- 需要 Agent 执行能力：目前主要靠海外产品（Codex、Claude Code），通过 API 中转或协助充值访问
- 开发者：[ZeoAPI](https://www.zeoapi.com/register?aff=FM4J) 提供 OpenAI 兼容接口，国内直连

## 趋势判断

短期看，国内 Agent 产品会经历一轮"先收再放"。长期看，这个方向不会停——OpenAI 98% 员工都在用 Codex agent（OpenAI 官方数据），agent 是确定性的下一步。

国内只是节奏不同：先收紧、再规范、再开放。
